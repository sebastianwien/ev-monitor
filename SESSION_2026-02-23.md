## 📋 Was heute passiert ist (chronologisch)

### 1. Security Hardening (Phase 1-2)
- Hardcoded Secrets aus Code entfernt (JWT, DB-Passwörter)
- Environment Variables System implementiert (`.env`, `.env.example`)
- `.gitignore` erstellt/updated (verhindert .env commits)
- CORS auf `ALLOWED_ORIGINS` env var umgestellt
- SQL Logging deaktiviert (`show-sql: false`)

### 2. OAuth2 Debugging (Phase 2-3)
**Problem:** Backend crash loop mit `IllegalStateException: Client id must not be empty`
- **Root Cause:** OAuth2 Config in `application.yml` mit leeren env vars
- **Fix 1:** OAuth2 Config in `application-oauth.yml` ausgelagert
- **Fix 2:** OAuth2 Handler aus `SecurityConfig` entfernt
- **Result:** Backend startet ohne OAuth2, kann später optional aktiviert werden

### 3. Server Deployment (Phase 4)
- Hetzner Server Setup: User `ihle`, Docker installiert, Firewall konfiguriert
- Code auf Server deployed (`/opt/ev-monitor/ev-monitor`)
- `.env` konfiguriert mit Secrets
- Docker Compose gestartet

### 4. SSL/Let's Encrypt Setup (Phase 5)
**Problem 1:** Backend crash loop → OAuth2 fix (siehe oben)

**Problem 2:** Let's Encrypt gibt 404 bei ACME Challenge
- Debugging: Volume mounts OK, nginx config OK, test file erstellt
- `curl http://ev-monitor.net/.well-known/acme-challenge/test.txt` → **Apache 404?!**

**Problem 3: THE BIG ONE** 🔥
- **Discovery:** `curl` zeigt Apache, aber `netstat` zeigt docker-proxy auf Port 80
- **Deep Dive:** DNS check, traceroute, alles zeigt richtigen Server
- **EUREKA:** DNS hat **IPv6 AAAA Record** auf FALSCHEN Server!
  ```
  IPv4: 46.225.210.231  ✅ (richtiger Server)
  IPv6: 2a01:4f8:d0a:27bd::2  ❌ (alter Server mit Apache!)
  ```
- **Fix:** AAAA Records im Hetzner DNS Panel korrigiert
- **Result:** Let's Encrypt konnte endlich auf richtigen Server zugreifen!

### 5. SSL Certificate erfolgreich geholt
```bash
docker compose run --rm --entrypoint certbot certbot \
  certonly --webroot \
  --webroot-path=/var/www/certbot \
  -d ev-monitor.net -d www.ev-monitor.net \
  --email sebastian.wien@posteo.de \
  --agree-tos --no-eff-email --non-interactive
```

### 6. Nginx mit HTTPS aktiviert
```bash
docker compose exec nginx nginx -s reload
```

### 7. Public Endpoints freigeben
**Problem:** `curl https://ev-monitor.net/api/cars/brands` → 403 Forbidden

**Fix:** SecurityConfig angepasst:
```java
.requestMatchers("/api/cars/brands", "/api/cars/brands/**").permitAll()
.requestMatchers("/api/vehicle-specifications/lookup").permitAll()
```

**Result:** API Endpoints für Registration Flow jetzt public!

## 🎓 Lessons Learned

1. **IPv6 DNS ist tricky!** Immer AAAA Records checken, nicht nur A-Records
2. **Let's Encrypt nutzt IPv6** wenn verfügbar → AAAA muss stimmen!
3. **Spring Security + OAuth2** sind nicht optional wenn configured → crasht bei leeren vars
4. **Docker Volume Mounts** sharen state zwischen Containern (certbot ↔ nginx)
5. **Nginx Template System** ersetzt `${DOMAIN}` automatisch aus env var
6. **Fail-Fast ist gut:** Keine Default-Werte für Secrets = App crasht früh statt später
